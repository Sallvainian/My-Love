## Key Security Fixes

1. **RLS recursion** (#11): Partner lookup in RLS policies caused infinite recursion. Fixed by using `SECURITY DEFINER` function.
2. **Privilege escalation** (#7): Users could set arbitrary `partner_id` via UPDATE. Fixed by restricting updatable columns.
3. **Search path injection** (#16): Functions without explicit `search_path` could be exploited. Fixed by adding `SET search_path = ''` to all functions.

## Schema Evolution Patterns

- **Additive changes:** New columns, new tables, new functions (most migrations)
- **Security patches:** RLS policy rewrites, function security hardening
- **Function rewrites:** `CREATE OR REPLACE FUNCTION` for iterative RPC improvements (lock-in function has 3+ revisions)
- **Phase guards:** Progressively tighter validation added to RPCs to prevent invalid state transitions
