## User Story

**As a** test engineer,
**I want** observable subscription health signals exposed from Supabase Realtime connections,
**So that** I can write deterministic E2E tests for subscription reconnection and message delivery.

---

## Context & Background

### Why This Story Exists

During test design analysis for TD-1.2 (Love Notes E2E Regeneration), Risk R-001 was identified:

> **R-001 (Score 6):** Supabase Realtime subscription drops silently, missing messages

To write deterministic E2E tests that verify subscription reconnection behavior, we need observable signals that tests can hook into. Currently, subscription drops are silent - there's no way to assert on connection state or reconnection events.

### Discovery Source

- **Workflow:** Party Mode discussion with TEA (Test Engineering Architect)
- **Course Correction:** `docs/05-Epics-Stories/sprint-change-proposal-2025-12-07.md`

### Problem Statement

Without subscription observability:
- Tests must use timeout-based waits (violates `e2e-quality-standards.md`)
- Subscription drop detection is non-deterministic
- Flaky tests undermine the 85/100 quality target
- R-001 risk remains unmitigated

## Acceptance Criteria

### AC1: useSubscriptionHealth Hook Created

**Given** a component using Supabase Realtime subscriptions
**When** the `useSubscriptionHealth` hook is called
**Then** it should expose:
- `connectionState`: 'connecting' | 'connected' | 'disconnected' | 'reconnecting'
- `lastHeartbeat`: Date | null
- `reconnectionCount`: number
- `isHealthy`: boolean (derived: connected && heartbeat within threshold)

**Verification:**
```typescript
const { connectionState, lastHeartbeat, reconnectionCount, isHealthy } = useSubscriptionHealth();
expect(connectionState).toBe('connected');
expect(isHealthy).toBe(true);
```

### AC2: Love Notes Subscription Integrated

**Given** the `useLoveNotesSubscription` hook
**When** subscription health is queried
**Then** it should reflect the underlying channel's connection state

**Verification:**
```typescript
const { subscriptionHealth } = useLoveNotesSubscription();
expect(subscriptionHealth.connectionState).toBe('connected');
```

### AC3: Connection State Transitions Observable

**Given** a Supabase Realtime subscription
**When** the connection drops and reconnects
**Then** the state transitions should be observable:
1. `connected` -> `disconnected` (on drop)
2. `disconnected` -> `reconnecting` (on reconnect attempt)
3. `reconnecting` -> `connected` (on successful reconnect)

**Verification:**
```typescript
// E2E test can wait for specific state
await page.waitForFunction(() =>
  window.__subscriptionHealth?.connectionState === 'reconnecting'
);
```

### AC4: Test Fixtures Support Mocking

**Given** the E2E test setup for Love Notes
**When** tests need to simulate subscription drops
**Then** fixtures should support:
- Mocking connection state
- Triggering reconnection events
- Asserting on state transitions

**Verification:**
```typescript
// In love-notes.setup.ts
await mockSubscriptionDrop(page);
await expect(page.getByTestId('connection-status')).toHaveText('Reconnecting...');
await mockSubscriptionReconnect(page);
await expect(page.getByTestId('connection-status')).toHaveText('Connected');
```

### AC5: No Production UI Required

**Given** the observability infrastructure
**When** used in production
**Then** UI indicators are OPTIONAL (not required for this story)

**Note:** A connection status indicator in Love Notes UI is a nice-to-have enhancement but not required for this story. The primary goal is test infrastructure.

  page: Page,
  state: ConnectionState

  );
}

## Dev Notes

### Supabase Channel States

The Supabase RealtimeChannel emits these states:
- `SUBSCRIBED` - Successfully subscribed
- `TIMED_OUT` - Subscription timed out
- `CLOSED` - Channel closed
- `CHANNEL_ERROR` - Error occurred

### Window Exposure Pattern

For E2E testing, expose state to window in dev mode:
```typescript
if (process.env.NODE_ENV !== 'production') {
  (window as any).__subscriptionHealth = health;
}
```

This allows Playwright to query state without modifying production code.
