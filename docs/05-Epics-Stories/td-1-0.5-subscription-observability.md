# Story TD-1.0.5: Subscription Observability Infrastructure

Status: done

## Story

As a test engineer,
I want observable subscription health signals exposed from Supabase Realtime connections,
so that I can write deterministic E2E tests for subscription reconnection and message delivery.

## Acceptance Criteria

1. `useSubscriptionHealth` hook created exposing: connectionState, lastHeartbeat, reconnectionCount, isHealthy
2. Love Notes subscription integrated with health hook
3. Connection state transitions observable (connected → disconnected → reconnecting → connected)
4. E2E test fixtures support mocking subscription drops and reconnects
5. No production UI required (test infrastructure only)
