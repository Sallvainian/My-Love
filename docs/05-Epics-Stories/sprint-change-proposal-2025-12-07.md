## Executive Summary

During test design analysis for TD-1.2, a technical gap was identified: **Supabase Realtime subscriptions can drop silently (Risk R-001, Score 6)**, and we lack the infrastructure to write deterministic E2E tests for subscription reconnection behavior.

**Proposed Action:** Add story TD-1.0.5 (Subscription Observability Infrastructure) as a prerequisite to TD-1.2.

---

## Identified Issue

### Problem Statement

The TD-1.2 story requires writing E2E tests for realtime messaging (Story 2-3: Real-Time Message Reception). Risk R-001 identifies that Supabase Realtime subscriptions can drop silently. To write deterministic E2E tests that verify subscription reconnection and message delivery after drops, we need **observable subscription health signals** (connection state, heartbeat, reconnection count). This infrastructure doesn't exist in the current codebase.

### Evidence

**From test-design-epic-2-love-notes.md:**
```
| R-001 | TECH | Supabase Realtime subscription drops silently, missing messages | 2 | 3 | 6 |
```

**From TEA Analysis (Party Mode):**
> "R-001 is your canary in the coal mine. Silent failures are testing poison. You can't assert on what you can't observe. We need observable subscription health - heartbeats, reconnection events, something the test can hook into."

### Impact Without Resolution

- Tests for R-001 mitigation would rely on timeout-based waits (violates `e2e-quality-standards.md`)
- Subscription drop detection would be non-deterministic
- Flaky tests would undermine the 85/100 quality target
- R-001 risk remains unmitigated in production
