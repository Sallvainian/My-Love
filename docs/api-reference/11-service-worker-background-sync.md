# 11. Service Worker & Background Sync

**Sources:**

- `src/sw.ts` -- Custom service worker with Workbox + Background Sync
- `src/sw-db.ts` -- IndexedDB helpers for service worker context
- `src/utils/backgroundSync.ts` -- Client-side Background Sync registration

## Service Worker (`src/sw.ts`)

### Caching Strategies

| Resource          | Strategy           | Cache Name         | Details                            |
| ----------------- | ------------------ | ------------------ | ---------------------------------- |
| JS/CSS            | `NetworkOnly`      | -                  | Always fresh after deployments     |
| Navigation (HTML) | `NetworkFirst`     | `navigation-cache` | 3s timeout, falls back to precache |
| Images/Fonts      | `CacheFirst`       | `static-assets-v2` | 100 entries, 30 days               |
| Google Fonts      | `CacheFirst`       | `google-fonts-v2`  | 30 entries, 1 year                 |
| Precache manifest | `precacheAndRoute` | -                  | Generated by VitePWA               |

### Background Sync Flow

1. `sync` event fires with tag `sync-pending-moods`
2. Opens IndexedDB via `sw-db.ts` helpers
3. Gets pending (unsynced) moods from `moods` store
4. Gets auth token from `sw-auth` store
5. Checks token expiry (5-minute buffer)
6. For each mood, calls Supabase REST API directly via `fetch()`
7. Marks synced moods in IndexedDB
8. Notifies open clients via `postMessage({ type: 'BACKGROUND_SYNC_COMPLETED' })`
9. If all fail, throws to trigger retry via Background Sync API

### Mood Transformation

Handles both single-mood (legacy) and multi-mood entries:

- `mood_type`: primary mood
- `mood_types`: array of all selected moods

## SW-DB Helpers (`src/sw-db.ts`)

### `getPendingMoods(): Promise<StoredMoodEntry[]>`

Opens DB, filters moods where `synced === false`. Closes DB after.

### `markMoodSynced(localId: number, supabaseId: string): Promise<void>`

Sets `synced: true` and `supabaseId` on the mood entry.

### `storeAuthToken(token): Promise<void>`

Stores auth token in `sw-auth` store with key `'current'`. Called by `sessionService.onAuthStateChange()`.

### `getAuthToken(): Promise<StoredAuthToken | null>`

Reads current auth token. Used by SW for API authentication.

### `clearAuthToken(): Promise<void>`

Deletes auth token on sign-out.

## Client-Side Registration (`src/utils/backgroundSync.ts`)

### `registerBackgroundSync(tag: string): Promise<void>`

Registers Background Sync tag via `registration.sync.register(tag)`. Guards for API support.

### `setupServiceWorkerListener(onSyncCompleted): () => void`

Listens for `BACKGROUND_SYNC_COMPLETED` messages from SW. Returns cleanup function.

### `isBackgroundSyncSupported(): boolean`

Checks for both `serviceWorker` and `SyncManager` in window.

### `isServiceWorkerSupported(): boolean`

Checks for `serviceWorker` in navigator (more permissive).
