## Team Discussion

### What Went Well

- **`togetherMode` fixture** — Encapsulated entire multi-browser lifecycle (seed, link, navigate, cleanup). Every Together Mode E2E test plugged into it. Compound reuse across all 3 stories.
- **Network-first pattern** — Zero `waitForTimeout` or `sleep` calls in the entire Epic 4 test suite. Every RPC interaction uses `waitForResponse` with typed predicates. Near-zero flakiness.
- **Client-side broadcast decision** — Discovered early that `PERFORM realtime.send()` doesn't work in local Docker Supabase. Moved all broadcasts client-side via `channel.send()` after successful RPC calls. Documented in project-context.md.
- **DB + Zustand injection shortcut** (E2E-004) — Jump to step 17 via `supabaseAdmin.update()` + `page.evaluate()` instead of 16 lock-in cycles. Kept test within 90s timeout.
- **Test quality improvement arc** — Story 4.3 went from 78/100 (C+) to 99/100 (A) after refactoring. Suite average improved from 88 to 97/100.
- **100% follow-through on Epic 3 action items** — All 3 items completed, all lessons applied. pgTAP couple relationship checklist prevented "tests passing by coincidence" failure mode.
- **First real-time feature shipped** — Supabase Broadcast, Presence channels, optimistic concurrency — none existed before Epic 4. Shipped with zero production incidents.

### Challenges

- **CI/infrastructure friction** — ~12 commits were purely CI fixes (ES256 JWT re-signing, burn-in job bugs, test pipeline, bundle-size workflow, CodeQL). None related to the actual feature.
- **ES256 JWT issue** — Supabase CLI v2.71.1+ silently changed GoTrue to ES256 signing, but `supabase status -o env` outputs stale HS256-signed keys. Required extracting ES256 private key from Docker container and adding re-signing logic to `playwright.config.ts`.
- **Mid-epic secrets migration** — Doppler -> dotenvx -> fnox/age. 5+ commits of infrastructure churn during active feature development.
- **Dev agent not following ATDD patterns** — The ATDD checklist specified existing helpers to reuse, but the dev agent duplicated ~120 lines of shared helper logic inline. Led to Story 4.3 scoring 50/100 on maintainability initially.

---

## Key Insights

1. **Network-first is the standard** — Zero hard waits in Epic 4 tests. `waitForResponse` with typed predicates replaces all `waitForTimeout`/`sleep` patterns. This is now mandatory for all E2E tests.

2. **Multi-browser E2E fixtures enable Together Mode testing** — The `togetherMode` fixture pattern (seed, link, navigate, cleanup in `finally`) is reusable for any partner-interaction feature.

3. **Client-side broadcast is required for local Supabase** — `PERFORM realtime.send()` doesn't work in Docker. All broadcasts must go through `channel.send()` after successful RPC calls.

4. **AI agent compliance with established patterns is the bottleneck** — The TEA module produces solid ATDD checklists with correct helper references. The dev agent doesn't consistently follow them. This is an AI behavior limitation, not a process gap.

5. **CI infrastructure requires ongoing maintenance** — Supabase CLI updates, JWT signing changes, and dependency updates create friction that's unrelated to feature work but consumes significant development cycles.

6. **Test quality can recover dramatically** — Story 4.3 went from 78 (C+) to 99 (A) with focused refactoring. Inline helpers extracted, conditional flow removed, file reduced 383->262 lines.
