Countdown Screen

Verse Screen — Reader Role

The reader sees their verse with a clear indicator. Playfair Display makes scripture feel reverent, not like a text doc.

Lock-in Button — All 4 States

The "both must confirm" pattern ensures synchronized progression. Neither partner can rush the other.

  • Available: Primary gradient — "Ready for next verse"
  • You're Ready: Secondary outlined — "Ready ✓ Tap to undo"
  • Sync Paused: Muted accent — "Holding your place"
  • Both Ready: Fading out — "Continuing..." (300-400ms transition)
No-blame copy: Primary helper text is "We'll continue when you're both ready" — never "Waiting for Jordan..." as primary (that subtly makes partner the bottleneck).

End-of-Session Reflection

After all 17 verses, users capture their thoughts. This is where vulnerability is invited, not demanded.

  • Only bookmarked verses shown (not all 17) — reduces overwhelm
  • No bookmarks? "You didn't mark any verses — that's okay"
  • Rating: 1-5 with "A little" ↔ "A lot" labels
  • Required: verse selection + rating. Optional: note + message
  • Bookmark sharing toggle: private by default, opt-in
  • Validation: quiet helper text on "Continue" tap, no red flashes
Ritual language: "Share reflection" not "Submit". "Close for today" not "Done". These words matter for emotional safety.

Solo Mode

Self-paced reading without synchronization. No partner indicator, no lock-in — just you and the text.

  • No role indicator (no partner to coordinate with)
  • No partner position dot
  • "Next Verse" instead of "Ready for next verse"
  • Bookmark flag still available — surfaces at reflection
  • Progress saves automatically (IndexedDB + server sync)
  • Resumable: "Continue from Verse 5" on next visit
  • Optimistic UI — changes appear instant, server syncs in background
Save/Resume: Auto-save is invisible. On return, user sees "Continue from Verse X" or "Start fresh" (preserves agency).

Reconnection Flow

Graceful handling when a partner's connection drops mid-session. Calm patience, not alarm.

  • "Jordan is reconnecting..." with subtle spinner
  • "You can keep reading" — reassures the user
  • 30-second timeout before showing "End for today" option
  • Server-authoritative state ensures clean resync
  • "End for today" not "End session" — gentle language
  • Both partners' progress saved on clean exit
Error emotion: Calm patience, not alarm. Network drops → neutral, presence-aware waiting. Never problem-focused.
